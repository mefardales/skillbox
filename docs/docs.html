How Skills Work

A skill is a Markdown file that contains expert-level instructions, patterns, and examples for a specific technology. When installed, the AI coding tool loads the skill file and uses its contents to inform every response it generates.

Skills are not documentation — they are written specifically for AI coding assistants. Instead of explaining concepts, they tell the AI exactly how to structure code, which patterns to prefer, and which anti-patterns to avoid.

Skills are plain Markdown files with YAML frontmatter. They are human-readable, version-controlled, and composable. You can install multiple skills for a single project.

Lifecycle of a Skill

Here is what happens when you run npx skillbox install frontend/react-components:

  1. Skillbox fetches the skill from the registry (or your local registry).
  2. It scans your home directory for known AI tool directories.
  3. The SKILL.md file is written to each tool's skills folder.
  4. On the AI tool's next startup, the skill is automatically loaded into context.
  5. The AI assistant uses the skill's instructions when generating code.

Global vs. Project Skills

By default, skills install globally to ~/.<tool>/skills/ and apply to all projects. Use the --project flag to install locally to ./.<tool>/skills/ in your current working directory. Project-level skills take precedence over global ones.

Global vs. project install 
# Global (default) — applies to all projects
npx skillbox install frontend/react-components

# Project-local — only for this project
npx skillbox install frontend/react-components --project

Install Paths

Skillbox installs skills to the standard path for each AI tool. These paths are defined by the Agent Skills specification and are recognized automatically by all compatible tools.

Tool Global path Project path
Claude Code ~/.claude/skills/ ./.claude/skills/
Cursor ~/.cursor/skills/ ./.cursor/skills/

Run npx skillbox detect to see exactly which tools are installed on your system and confirm the paths that will be used.

Compatible Tools

Skillbox works with any AI coding tool that supports the Agent Skills open standard. The standard defines a simple directory convention — any tool that reads a skills/ directory from the tool's config folder will work.

Officially Supported

Claude Code — Anthropic's official CLI assistant
Cursor — AI-first code editor
Codex — OpenAI's coding model CLI
Windsurf — Codeium's AI editor
Aider — AI pair programming in the terminal
Continue.dev — Open-source AI code assistant for VS Code / JetBrains

See the full list on the Agent Skills website. 30+ tools are compatible with the standard.

Skill Format (SKILL.md)

Every skill is a single Markdown file named SKILL.md. It contains a YAML frontmatter block followed by the skill content in Markdown.

Frontmatter Fields

Field Type Required Description
name string Yes Unique skill identifier (kebab-case, matches directory name)
description string Yes One-sentence description of what the skill teaches the AI
category string Yes One of: backend, data, devops, frontend, general, testing
tags string[] Yes 3–8 searchable keywords (lowercase, no spaces)
version string No Semantic version of the skill (e.g. 1.0.0). Defaults to 1.0.0.
authors string[] No GitHub usernames of skill authors

Example SKILL.md

skills/frontend/react-components/SKILL.md 
---
name: react-components
description: React component best practices with hooks, TypeScript, and modern patterns.
category: frontend
tags: [react, typescript, hooks, components, frontend]
version: 1.2.0
authors: [your-github-username]
---

# React Component Best Practices

## Component Structure

Always prefer functional components with hooks over class components...

## Naming Conventions

- Components: PascalCase (e.g. UserProfile)
- Hooks: camelCase prefixed with 'use' (e.g. useUserProfile)
- Props interfaces: ComponentNameProps (e.g. UserProfileProps)

Content Guidelines

The Markdown body should contain actionable instructions — not just information:

  • Write instructions, not explanations. Tell the AI what to do, not what a concept means.
  • Use concrete code examples for every pattern you describe.
  • Specify what to prefer and what to avoid explicitly.
  • Keep the skill focused on one technology — do not try to cover everything.
  • Assume the AI already knows the basics — focus on expert-level patterns.

Agent Skills Standard

Skillbox is built on the Agent Skills open standard — a simple, tool-agnostic convention for distributing reusable AI instructions.

The standard defines:

  • The directory structure where skills are stored (~/.<tool>/skills/)
  • The SKILL.md file format with YAML frontmatter
  • How tools should discover and load skills at startup
  • Registry API conventions for skill distribution
By following the open standard, any skill you create in Skillbox will automatically work in all 30+ compatible tools — no per-tool integration needed.

Read the full specification at agentskills.io/spec or visit the Agent Skills website.
